### 3.4 关键技术实现细节

#### 3.4.1 自由语音文本输出

```java

```

## 4. 开发注意事项

1. **日志匹配精度**：所有事件必须使用精确字符串包含匹配，注意空格和大小写
2. **反向逻辑**：近光灯状态为反向逻辑（0=开，1=关），开发时必须注意转换
3. **重复事件**：同一状态变化可能会从多个TAG输出，建议做防抖处理（300ms内只处理一次）
4. **权限要求**：监听Logcat需要`android.permission.READ_LOGS`权限；发送系统广播需要零跑专属广播权限
5. **兼容性**：不同车机系统版本的日志格式可能略有差异，建议增加容错处理
